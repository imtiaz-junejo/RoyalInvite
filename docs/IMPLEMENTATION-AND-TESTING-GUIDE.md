# Implementation And Testing Guide

This document summarizes what has been implemented in `Eternally Yours` and how to test it end to end.

## Overview

The project now includes:
- core wedding invitation builder
- public invitation microsite experience
- multilingual content support
- story gallery and love story sections
- wedding schedule and multi-event support
- guestbook / wishes wall
- enhanced RSVP flow
- event-day guest check-in mode
- invitation view analytics
- tier-based product restrictions
- production Docker deployment flow

## Implemented Features

### 1. Core Invitation Builder
- create invitation
- choose template preset
- choose theme
- choose font
- add venue, date, time, maps link
- add invitation message
- add RSVP details
- add music URL

Routes:
- `/dashboard/invitations/new`
- `/dashboard/invitations/[id]/edit`

### 2. Wedding Microsite Features

#### Love Story
- story title
- story text
- secondary story title
- secondary story text

#### Multilingual Invite Support
- primary language
- optional secondary language
- language toggle on public invite page

Supported languages:
- English (`en`)
- Hindi (`hi`)
- Urdu (`ur`)
- Marathi (`mr`)
- Gujarati (`gu`)

#### Wedding Day Schedule
- same-day schedule section
- line format:

```text
4:00 PM | Guests Arrive
4:30 PM | Ceremony
6:00 PM | Dinner & Dancing
```

#### Multi-Event Wedding Support
- separate event-series section for wedding week functions
- line format:

```text
Mehndi | 12 Dec 2026 | 5:00 PM | Jasmine Courtyard
Sangeet | 13 Dec 2026 | 7:00 PM | Sapphire Hall
Wedding | 14 Dec 2026 | 11:00 AM | Noor Palace
```

#### Story Gallery
- gallery entries with image URL and caption
- line format:

```text
https://image1.jpg | Our engagement day
https://image2.jpg | A favorite memory together
```

### 3. Guest-Facing Invite Experience
Public route:
- `/invite/[slug]`

Public invite now includes:
- animated hero names
- countdown timer
- music player
- 3D invitation card
- venue section
- love story
- story gallery
- wedding day schedule
- multi-event timeline
- enhanced RSVP form
- guestbook / wishes wall
- share modal
- language switcher when a second language exists

### 4. Enhanced RSVP
RSVP now supports:
- guest name
- guest email
- attendance status
- guest count
- meal preference
- message
- autogenerated check-in code

After RSVP submission:
- guest sees confirmation state
- guest check-in code is shown

### 5. Guestbook / Wishes Wall
Guests can:
- submit their name
- submit a short blessing/wish

Hosts can see public entries directly on the invite page.

### 6. Event-Day Guest Check-In
Dashboard route:
- `/dashboard/invitations/[id]/check-in`

Supports:
- search guests by name
- search/check in by code
- one-click row-based check-in
- checked-in state persistence

### 7. Invitation Analytics
Tracked now:
- invitation views
- RSVP counts
- guestbook entry counts

Dashboard shows:
- total views across invitations
- per-invitation views
- per-invitation wishes

### 8. South Asian Template Presets
New presets:
- `Mehndi Garden`
- `Noor Nikah`
- `Heritage Royale`

These presets prefill:
- story content
- meal options
- schedules
- event-series data
- multilingual defaults

### 9. Free / Pro Tier Rules

#### Free Tier
- can create up to `5` invitations
- cannot edit invitations after creation

#### Pro Tier
- unlimited invitations
- can edit invitations after creation

Enforced in:
- create API
- update API
- dashboard UI
- edit route access

## Important Business Rules

### Free Tier Editing Rule
Once a free user creates an invitation:
- they can still view it
- they can still publish it if creation flow allows it at that point
- they cannot later reopen and edit it

### Pro Tier Editing Rule
Pro users can:
- create
- save draft
- publish
- reopen
- edit later

## Local Testing

### 1. Install and prepare
```bash
npm install
cp .env.example .env.local
```

Make sure `.env.local` has:

```env
DATABASE_URL="file:./dev.db"
NEXTAUTH_SECRET="your-secret"
NEXTAUTH_URL="http://localhost:3000"
APP_URL="http://localhost:3000"
```

### 2. Sync database and generate Prisma client
```bash
set -a && . ./.env.local && set +a
npx prisma generate
npx prisma db push --accept-data-loss
```

### 3. Seed admin user if needed
```bash
npx tsx prisma/seed.ts
```

Default admin credentials:
- Email: `admin@sikanderkumbhar.com`
- Password: `Admin@123456`

### 4. Run app
```bash
npm run dev
```

### 5. Build validation
```bash
npm run build
```

## Functional Test Checklist

### A. Auth

#### Sign In
1. Open `/sign-in`
2. Use admin credentials
3. Confirm redirect to `/dashboard`

Expected:
- successful login
- no auth error loop

#### Sign Up
1. Open `/sign-up`
2. Create a temporary user
3. Confirm redirect to `/dashboard`

Expected:
- account is created
- user session is active

Note:
- if sign-in behaves strangely after deploy, clear cookies for `invite.sikanderkumbhar.com`

### B. Free Tier Rule Test
1. Create a new free-tier user
2. Create invitation 1 through 5
3. Try creating invitation 6

Expected:
- first 5 succeed
- 6th is blocked

Then:
4. Try opening edit page for any created free-tier invitation

Expected:
- edit is blocked
- dashboard shows lock messaging

1. Set a user tier to `pro`
2. Create invitation
3. Reopen it in edit page

Expected:
- edit works normally

- secondary language
- meal options

## Known Notes

### Prisma runtime target
`schema.prisma` includes:
- `native`
- `linux-musl`
- `debian-openssl-3.0.x`

This is required because:
- build stage uses Alpine
- runtime container uses Debian

### Server disk space
The EC2 host previously hit Docker image pull disk-space issues.

Mitigations already added:
- `.dockerignore`

If needed on server:
```bash
docker image prune -af
docker system df
```
